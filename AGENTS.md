# Developer Tool Rules

These rules are mandatory for any coding agent or developer tool working in this repository.

1. Think before writing
- Do not guess. If any requirement is unclear, stop and ask for clarification first.

2. Simplicity first
- Implement only what is needed to solve the requested problem.
- Avoid unnecessary complexity, abstractions, or extra features.

3. Surgical intervention
- Change only the minimum required files and lines.
- Do not modify unrelated or already working code.

4. Purposeful implementation
- Base changes on clear tests or validation steps.
- Confirm the result actually achieves the requested goal.
